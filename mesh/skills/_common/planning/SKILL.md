# Planning Skill

## When to use
When the user requests a plan, roadmap, feature breakdown,
task estimation, or pre-implementation analysis.

## Process
1. Understand the end goal and constraints.
2. Decompose into atomic, independent tasks.
3. Identify dependencies between tasks.
4. Estimate complexity (S/M/L) per task.
5. Propose execution order and potential risks.

## Output format
- Plan title
- Objective
- Task list with estimates and dependencies
- Identified risks
- Concrete next step
