# Code Review Skill

## When to use
When the user asks to review code, a PR, a diff, or a specific function.

## Process
1. Analyze architecture and patterns used.
2. Verify error handling and edge cases.
3. Evaluate readability and maintainability.
4. Check security implications.
5. Suggest concrete improvements with code examples.

## Checklist
- [ ] Naming conventions followed
- [ ] Errors handled correctly
- [ ] Edge cases covered
- [ ] No obvious vulnerabilities
- [ ] Sufficient test coverage
