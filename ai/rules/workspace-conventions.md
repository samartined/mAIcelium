# Workspace Conventions

## Source of truth

All AI configuration lives in `ai/`. The dotfolders (`.cursor/`, `.claude/`,
`.antigravity/`) are **auto-generated mirrors** via symlinks. Never write
directly to them.

When creating or modifying any AI resource (rule, skill, command, prompt),
always target the canonical path under `ai/`.

## Naming

- Files and directories: **kebab-case** (e.g., `code-review`, `security-checklist.md`).
- Project-prefixed symlinks use `--` as separator (e.g., `my-api--custom-rule.md`).

## Command output

Slash command responses must end with a single emoji-prefixed status line:
- ✅/✔ success — ⚠️ warning — ❌ error — ❓ ambiguous — 🗑️ removed — 📭 empty

## File placement quick reference

| What              | Where                                  |
|-------------------|----------------------------------------|
| Global rule       | `ai/rules/<name>.md`                  |
| Universal skill   | `ai/skills/_common/<name>/SKILL.md`   |
| Domain skill      | `ai/skills/_domains/<name>/SKILL.md`  |
| Client skill      | `ai/skills/_clients/<name>/SKILL.md`  |
| Command           | `ai/commands/<name>.md`               |
| Script            | `ai/commands/scripts/<name>.py`       |
| Prompt template   | `ai/prompts/<name>.md`                |

## Auto-generated files (do not edit)

- `WORKSPACE.md` — updated by `bin/add_project.sh` / `bin/remove_project.sh`
- `.claude/projects-context.md` — regenerated by `bin/_lib.sh` (inlines workspace + project rules)
- `.cursor/rules/*` — symlinks created by `bin/init.sh` / `bin/sync_symlinks.sh`
- `.cursor/skills-cursor/*` — symlinks created by `bin/init.sh` / `bin/sync_symlinks.sh`
- `.antigravity/rules` / `.antigravity/skills` — symlinks
